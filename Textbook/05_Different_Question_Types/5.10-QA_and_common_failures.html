Most WeBWorK authoring time is not spent on the "main math", but on the predictable failure modes: an answer blank is not bound to the intended grader, a unit or format expectation is unclear, or randomization generates a broken biology scenario. This section collects a small set of checks that catch most failures quickly.

Answer blank binding and ordering

  • Prefer attaching answers directly to blanks in PGML: [_____]{...}.
  • For multi-part prompts, keep one target quantity per blank.
  • Preview the problem and confirm each blank grades the intended thing.

Text answers (biology terminology) that cause grading disputes

  • Decide what you will accept for plural/singular (mitochondrion/mitochondria).
  • Decide how you will treat hyphens and spacing (Gram-positive vs Gram positive).
  • If abbreviations are acceptable, include them explicitly (DNA vs deoxyribonucleic acid).
  • For gene/protein names, state whether case matters and whether symbols or full names are expected.
  • When spelling is not the point, use recognition-style items (MC/MATCH) instead of free text.

Units and reporting expectations

  • State units in the prompt and repeat the expected units next to the answer blank.
  • Use one unit system per question whenever possible.
  • State rounding or decimal-place expectations when they matter for grading.

Tolerances and "nearly correct" answers

  • Set tolerances intentionally (absolute vs relative) and keep them consistent with your reporting rules.
  • Test boundary cases: values just inside/outside tolerance, common rounding choices, and common unit mistakes.
  • When students keep retrying, the issue is often unclear expectations rather than hard content.

Randomization and biological plausibility

  • Constrain random values to avoid impossible scenarios (negative concentrations, invalid probabilities, etc.).
  • Avoid values that force awkward repeating decimals unless you want to grade with tolerance.
  • Preview multiple variants and spot-check the story, units, and answer reasonableness.

Log-scale traps (pH, Ct, fold change)

  • State what quantity you want (for example pH vs [H+]).
  • State the model and base when needed (for example Ct differences under a 2x-per-cycle assumption).
  • State the reporting format (Ct, delta Ct, fold change, log2 fold change) to avoid multiple interpretations.
