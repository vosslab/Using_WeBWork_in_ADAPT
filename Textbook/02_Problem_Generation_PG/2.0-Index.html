WeBWorK problems are written in PG, but in this guide the goal is not to turn instructors into Perl programmers. Regular PG is treated as the minimum setup needed to support a clean PGML prompt: define a few variables, define answer checkers, and let PGML carry the student-facing text.

What "PG" is used for (in this guide)

  • Load macros needed for PGML and answer checking.
  • Choose an appropriate MathObjects context.
  • Generate a small number of randomized values.
  • Compute the target answer(s) and configure the answer checkers.

What "regular PG" is not used for (in this guide)

  • Writing the prompt as long blocks of BEGIN_TEXT/END_TEXT.
  • Building complex formatting by hand (PGML is preferred).
  • Large "programs" inside a single question.

How this chapter fits the book

Chapter 2 exists because every PGML problem still needs some PG scaffolding (macros, variables, answers). The goal is to keep that scaffolding short, readable, and reusable.
